is_list [Reduce]
Decide if the rule is a list, that is if it has the following shape: 
l(xs) ::=           or    l(xs) ::=
        | eps                     | `cons`
        | `cons`                  | eps
where `cons` = s(ys) :: acts, s and l are equivalent according to Reduce.alias, xs and ys are equivalent according to Reduce.equal_params, s does Reduce.not_occurs in acts

is_nonempty_list [Reduce]
Decide if the rule is a separated non empty list, that is if it has the following shape: 
nel(xs) ::=           or    nel(xs) ::=
          | `base`                    | `cons`
          | `cons`                    | `base`
where `cons` = s(ys) :: acts, s and nel are equivalent according to Reduce.alias, xs and ys are equivalent according to Reduce.equal_params, s does Reduce.not_occurs in acts, acts = `base` This condition is explained for a left-recursive list but the function also copes with right-recursion.
is_option [Reduce]
Decide if the rule is an optional, that is if it has the following shape: 
r ::=       or    r ::=
   | eps              | `some`
   | `some`           | eps
is_sep_list [Reduce]
Decide if the rule is a separated list, that is if it has the following shape: 
sl ::=              or    sl ::=
     | eps                     | snel(...)
     | snel(...)               | eps
where snel is recognized as a separated_nonempty_list by Reduce.is_sep_nonempty_list.
is_sep_nonempty_list [Reduce]
Decide if the rule is a separated non empty list, that is if it has the following shape: 
snel(xs) ::=           or    snel(xs) ::=
           | `base`                     | `cons`
           | `cons`                     | `base`
where `cons` = s(ys) :: acts, s and snel are equivalent according to Reduce.alias, xs and ys are equivalent according to Reduce.equal_params, s does Reduce.not_occurs in acts, acts = sep @ `base` This condition is explained for a left-recursive list but the function also copes with right-recursion.
